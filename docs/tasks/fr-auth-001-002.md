# FR-AUTH-001 / FR-AUTH-002 実装計画書

## 概要

VectorNote の認証機能（Googleログイン・セッション管理）を仕様通りに完成させる。

## 実装状況

### 実装済み

- [x] Firebase初期化・Auth設定 (`apps/web/src/lib/firebase.ts`)
- [x] GoogleAuthProvider + signInWithPopup によるGoogleログイン (`providers/FirebaseAuthProvider.tsx`)
- [x] onAuthStateChanged による認証状態監視 (`providers/FirebaseAuthProvider.tsx`)
- [x] ログインページUI (`routes/login.tsx`)
- [x] ログアウト機能 (signOut) (`routes/index.tsx`)
- [x] ルートレイアウトでのログインページ条件分岐 (`routes/__root.tsx`)
- [x] Sonner トーストコンポーネント (`components/ui/sonner.tsx`)

### 未実装

- [ ] TanStack Router ルートガード (`_authed.tsx` の beforeLoad)
- [ ] 認証済みルートの `_authed` 配下への移動
- [ ] 30日セッション有効期限チェック
- [ ] browserLocalPersistence の明示的設定
- [ ] ログインエラーのトースト表示
- [ ] ログアウト時の TanStack Query キャッシュクリア
- [ ] `<Toaster />` のルートレイアウトへの配置

## 実装手順

### Step 1: `<Toaster />` をルートレイアウトに配置

**ファイル:** `apps/web/src/routes/__root.tsx`

- `<Toaster />` を `<body>` 内に追加し、アプリ全体でトースト通知を利用可能にする

### Step 2: browserLocalPersistence を明示設定

**ファイル:** `apps/web/src/lib/firebase.ts`

- `setPersistence(auth, browserLocalPersistence)` を追加

### Step 3: `_authed.tsx` ルートガードの作成

**新規ファイル:** `apps/web/src/routes/_authed.tsx`

- `beforeLoad` で `auth.authStateReady()` を待機後、`auth.currentUser` を検証
- 未認証なら `/login` へリダイレクト

### Step 4: 既存ルートを `_authed` 配下に移動

- `routes/index.tsx` → `routes/_authed/index.tsx`
- `routes/about.tsx` → `routes/_authed/about.tsx`
- `routes/demo/tanstack-query.tsx` → `routes/_authed/demo/tanstack-query.tsx`

### Step 5: `login.tsx` に認証済みユーザーのリダイレクトを追加

**ファイル:** `apps/web/src/routes/login.tsx`

- `beforeLoad` で認証済みユーザーを `/` へリダイレクト

### Step 6: FirebaseAuthProvider のリダイレクトロジック削除

**ファイル:** `apps/web/src/providers/FirebaseAuthProvider.tsx`

- ルートガードがリダイレクトを担当するため、useEffect によるリダイレクトを削除
- `isAuthPath` を Provider から削除し、`__root.tsx` の AppLayout 内で直接判定に変更

### Step 7: 30日セッション有効期限チェック

**ファイル:** `apps/web/src/providers/FirebaseAuthProvider.tsx`

- ログイン成功時に `localStorage` にログイン日時を記録
- `onAuthStateChanged` 内で30日超過をチェックし、超過時は自動ログアウト

### Step 8: ログインエラーのトースト表示

**ファイル:** `apps/web/src/providers/FirebaseAuthProvider.tsx`

- `signInWithPopup` の catch で `toast.error()` を呼び出し
- ポップアップを閉じた場合（`popup-closed-by-user`）はエラー表示しない

### Step 9: ログアウト時のキャッシュクリア強化

**ファイル:** `apps/web/src/providers/FirebaseAuthProvider.tsx`

- `signOut` に加え、`localStorage` のセッション情報削除
- `queryClient.clear()` で TanStack Query キャッシュをクリア

## 検証方法

1. `pnpm dev` で開発サーバー起動、routeTree.gen.ts が正常に再生成されること
2. 未ログイン状態で `/` にアクセス → `/login` にリダイレクト
3. Googleログイン → `/` に遷移
4. ログイン後 `/login` にアクセス → `/` にリダイレクト
5. ログアウト → `/login` に遷移
6. ブラウザリロード後もログイン状態が維持
7. ログインエラー時にトーストが表示
8. `localStorage` に `auth_login_at` が保存されている
